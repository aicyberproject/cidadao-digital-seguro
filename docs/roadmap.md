# Roadmap

Este roadmap reúne pendências técnicas, editoriais e operacionais já identificadas para evolução do curso.

## Pendências

1. Videoaulas finais

   Produzir, revisar e integrar as videoaulas definitivas de todos os módulos, substituindo textos ou placeholders de preparação quando existirem.

2. Revisão de materiais complementares

   Revisar links oficiais, materiais permanentes e leituras complementares para garantir atualidade, pertinência institucional e disponibilidade pública.

3. Revisão final do certificado e versão

   Validar texto, layout, dados exibidos, código verificador simples e versão do curso antes de uma publicação institucional ampla.

4. Acessibilidade

   Revisar navegação por teclado, contraste, foco visível, semântica de botões, textos alternativos quando aplicável e leitura por tecnologias assistivas.

5. Revisão responsiva mobile

   Testar a jornada completa em telas pequenas, incluindo trilha lateral, quizzes, avaliação final, formulário de certificado e leitura dos módulos.

6. Validação institucional

   Submeter conteúdo, linguagem, fontes, orientações preventivas e fluxos de resposta a uma revisão institucional final.

7. Eventual backend/verificador público de certificado

   Avaliar, em etapa futura, se o certificado deve ter verificação pública com backend, banco de dados ou endpoint institucional. A versão atual gera código local sem validação remota.

## Critérios para priorização

Priorize mudanças que:

- reduzam risco de orientação incorreta;
- aumentem clareza para público leigo;
- preservem linguagem institucional e não alarmista;
- melhorem confiabilidade do fluxo principal;
- tenham validação por `npm run check`;
- mantenham escopo pequeno e revisável por Pull Request.
